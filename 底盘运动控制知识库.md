## 1. 运动函数速查

### LemLib 高级运动控制（Autonomous 专用）

| 函数 | 头文件 | 作用 |
|------|--------|------|
| `moveToPoint(target, timeout, params, settings)` | `lemlib/motions/moveToPoint.hpp` | 移动到 (x, y) 坐标，不关心最终朝向 |
| `moveToPose(target, timeout, params, settings)` | `lemlib/motions/moveToPose.hpp` | 移动到 (x, y) 并精确控制最终朝向 |
| `turnTo(target, timeout, params, settings)` | `lemlib/motions/turnTo.hpp` | 转向到绝对角度，或面向某个坐标点 |
| `follow(path, lookahead, timeout, params, settings)` | `lemlib/motions/follow.hpp` | 沿预设路径（asset 文件）跟踪行驶 |

**运动调度器**（`lemlib::motion_handler` 命名空间）：

| 函数 | 作用 |
|------|------|
| `motion_handler::move(fn)` | 执行运动 lambda，阻塞直到上一个运动结束 |
| `motion_handler::cancel()` | 取消当前正在执行的运动 |
| `motion_handler::isMoving()` | 检查是否有运动正在执行 |

**典型 autonomous 使用模式：**

```cpp
namespace ll = lemlib;
namespace mh = lemlib::motion_handler;

void autonomous() {
    mh::move([&] { ll::moveToPoint({0_in, 24_in}, 2_sec, params, settings); });
    mh::move([&] { ll::turnTo(90_stDeg, 1_sec, turnParams, turnSettings); });
    mh::move([&] { ll::moveToPose({24_in, 24_in, 180_stDeg}, 3_sec, poseParams, poseSettings); });
}
```

### 直接底盘控制（Opcontrol / 手动操控）

`lemlib::MotorGroup` API：

| 方法 | 说明 |

### 场景选择

| 场景 | 推荐函数 |
|------|---------|
| 自主开环走直线 | `moveToPoint` |

## 2. opcontrol() 手动操控

### 控制模型：Arcade Drive

```
左摇杆 Y 轴 → 前进/后退
右摇杆 X 轴 → 左转/右转

左轮 = dir - turn
右轮 = dir + turn
```

### 核心代码

```cpp
void opcontrol() {
    pros::Controller master(pros::E_CONTROLLER_MASTER);
    pros::MotorGroup left_mg({1, -2, 3});   // 端口号加负号 = 反转
    pros::MotorGroup right_mg({-4, 5, -6});

    while (true) {
        int dir  = master.get_analog(ANALOG_LEFT_Y);   // [-127, 127]
        int turn = master.get_analog(ANALOG_RIGHT_X);

        left_mg.move(dir - turn);
        right_mg.move(dir + turn);
        pros::delay(20);  // 50Hz 控制循环
    }
}
```

### 潜在改进

- **死区**：`if (abs(dir) < 10) dir = 0;` 防止摇杆中位噪声
- **钳位输出**：`std::clamp(dir - turn, -127, 127)` 防止越界
- **优化 LCD 读取**：`read_buttons()` 只调一次，存变量复用

## 3. moveToPoint() 定点移动

### 函数签名

```cpp
void moveToPoint(V2Position target, Time timeout,
                 MoveToPointParams params, MoveToPointSettings settings);
```

### 算法概要

1. **横向误差**：$lateralError = d \cdot \cos(\theta_{target} - \theta_{robot})$ — 只关心朝向目标方向上的距离
2. **角向误差**：机器人朝向 vs 指向目标的方向（支持 `reversed` 倒退模式）
3. **两阶段控制**：
   - **远距离**（> 7.5 英寸）：全速 + 角向修正
   - **沉降**（≤ 7.5 英寸）：限速 + 关闭角向修正
4. **PID 输出管道**：`PID → max钳位 → slew限加速度 → min钳位`
5. **desaturate()**：混合横向 + 角向，确保不超出电机能力
6. **双层退出**：标准沉降判断 + 越过目标线早退

### 参数说明

| 参数 | 类型 | 含义 |
|------|------|------|
| `target` | `V2Position` | 目标坐标 (x, y) |
| `timeout` | `Time` | 超时时间 |
| `params.maxLateralSpeed` | `Number` | 横向最大功率 |
| `params.maxAngularSpeed` | `Number` | 角向最大功率 |
| `params.minLateralSpeed` | `Number` | 最小功率（克服静摩擦） |
| `params.lateralSlew` | `Number` | 横向加速度限制 |
| `params.angularSlew` | `Number` | 角向加速度限制 |
| `params.reversed` | `bool` | 倒退行驶 |
| `params.earlyExitRange` | `Length` | 早退容忍距离 |
| `settings.lateralPID` | `PID` | 横向 PID |
| `settings.angularPID` | `PID` | 角向 PID |
| `settings.exitConditions` | `ExitCondition` | 沉降判断器 |
| `settings.poseGetter` | `function` | 位姿回调 |
| `settings.leftMotors` | `MotorGroup&` | 左电机组 |
| `settings.rightMotors` | `MotorGroup&` | 右电机组 |

## 4. moveToPose() 位姿移动

### 函数签名

```cpp
void moveToPose(Pose target, Time timeout,
                MoveToPoseParams params, MoveToPoseSettings settings);
```

### 核心创新：胡萝卜点 (Carrot Point)

用虚拟引导点引导机器人从目标朝向的延长线方向曲线入场。

```cpp
carrot = target - V2Position::fromPolar(target.orientation, lead × distance);
```

- 远距离：机器人追胡萝卜 → 绕弧线接近目标
- 沉降（< 7.5 英寸）：胡萝卜 = 目标本身，角向 PID 锁定 `target.orientation`

### moveToPose 独有的安全机制

| 机制 | 实现 | 作用 |
|------|------|------|
| **防侧滑** | 根据转弯半径计算 `maxSlipSpeed` | 急弯自动减速 |
| **角向优先** | `overturn` 削减横向功率 | 方向准确度优先于速度 |
| **方向保护** | `max(out, 0)` / `min(out, 0)` | 远距离绝不反向行驶 |

### 参数说明

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `reversed` | `false` | 倒退行驶 |
| `lead` | `0.6` | 胡萝卜引导系数 |
| `driftCompensation` | `drift_compensation` | 防侧滑系数 |
| `maxLateralSpeed` | `1` | 横向最大功率 |
| `minLateralSpeed` | `0` | 横向最小功率 |
| `maxAngularSpeed` | `1` | 角向最大功率 |
| `lateralSlew` | `lateral_slew` | 横向加速度限制 |
| `angularSlew` | `angular_slew` | 角向加速度限制 |
| `earlyExitRange` | `0_in` | 早退容忍距离 |

**Settings：**

| 字段 | 默认值 | 含义 |
|------|--------|------|
| `lateralPID` | `lateral_pid` | 横向 PID |
| `angularPID` | `angular_pid` | 角向 PID |
| `lateralExitConditions` | `lateral_exit_conditions` | 横向沉降判断 |
| `angularExitConditions` | `angular_exit_conditions` | 角向沉降判断 |
| `poseGetter` | `pose_getter` | 位姿回调 |
| `leftMotors` | `left_motors` | 左电机组 |
| `rightMotors` | `right_motors` | 右电机组 |

### 完整调用示例

```cpp
ll::moveToPose(
    {24_in, 24_in, 90_stDeg},  // 目标: (x, y, θ)
    3_sec,
    MoveToPoseParams { .lead = 0.6, .maxLateralSpeed = 0.9 },
    MoveToPoseSettings {}       // 全部使用全局默认值
);
```

## 5. moveToPose vs moveToPoint 对比

| 维度 | `moveToPoint` | `moveToPose` |
|------|---------------|--------------|
| **目标类型** | `V2Position` (x, y) | `Pose` (x, y, θ) |
| **最终朝向** | 不关心 | 精确到 `target.orientation` |
| **引导机制** | 无 | 胡萝卜点 |
| **横向误差** | $d \cdot \cos\Delta\theta$ | 远距离 $d \cdot \text{sgn}$，沉降 $d \cdot \cos$ |
| **角向目标** | 指向目标点 | 远距离指向胡萝卜，沉降锁定朝向 |
| **沉降角向** | 归零 | 持续工作，锁定朝向 |
| **功率混合** | `desaturate` 比例缩放 | 角向优先（overturn 削减横向） |
| **防侧滑** | 无 | 曲率半径限速 |
| **方向保护** | 无 | 严格单向 |
| **退出条件** | 1 维（横向） | 2 维（横向 + 角向） |

## 6. 坐标系说明

### 绝对坐标（场心坐标系）

- `target` 传入的是相对于里程计原点的**绝对场坐标**
- 里程计默认原点 = `(0_in, 0_in, 0_stDeg)` = 机器人起始位置
- 0° = X+ 方向

```
         Y+ (90°)
         ↑
    ─────◎────→ X+ (0°)
         |
```

### 如何传入相对偏移量

```cpp
void moveToPoseRelative(Pose offset, Time timeout, auto params, auto settings) {
    Pose current = settings.poseGetter();
    Pose absolute = {
        current.x + offset.x,
        current.y + offset.y,
        offset.orientation
    };
    ll::moveToPose(absolute, timeout, params, settings);
}
```

## 7. 里程计 & IMU 初始化

### 完整初始化代码

```cpp
#include "lemlib/lemlib.hpp"
#include "hardware/IMU/V5InertialSensor.hpp"
#include "lemlib/tracking/TrackingWheelOdom.hpp"

// 硬件对象（全局生命周期）
lemlib::V5InertialSensor imu(10);

lemlib::TrackingWheel verticalWheel(3, 2.75_in, 2_in);    // 纵向轮，端口3，偏右2英寸
lemlib::TrackingWheel horizontalWheel(-4, 2_in, -4_in);    // 横向轮，端口4(反)，偏后4英寸

lemlib::TrackingWheelOdometry odom(
    {&imu},                  // IMU 列表
    {&verticalWheel},        // 纵向追踪轮列表
    {&horizontalWheel}       // 横向追踪轮列表
);

void initialize() {
    // 1. 校准陀螺仪（必须保持静止）
    imu.calibrate();
    while (imu.isCalibrating()) { pros::delay(10); }

    // 2. 设置坐标系原点
    odom.setPose({0_in, 0_in, 0_stDeg});

    // 3. 启动后台更新任务（10ms 周期）
    odom.startTask(10_msec);
}
```

### 三步关键操作

| 步骤 | 代码 | 说明 |
|------|------|------|
| 校准 IMU | `imu.calibrate()` + 等待 | 机器人**必须静止**，约 2-3 秒 |
| 设置原点 | `odom.setPose({0_in, 0_in, 0_stDeg})` | 在 autonomous() 开头重置 |
| 启动任务 | `odom.startTask(10_msec)` | 独立 FreeRTOS Task，自动更新 |

### TrackingWheel 构造函数

| 传感器类型 | 示例 |
|-----------|------|
| V5 旋转传感器 | `TrackingWheel(3, 2.75_in, 2_in)` |
| ADI 编码器 | `TrackingWheel('A', 'B', false, 2_in, 2_in)` |
| ADI 扩展板 | `TrackingWheel(3, 'E', 'F', true, 2_in, 0_in)` |

### offset 符号约定

```
        +Y (前方)
         ↑
  (-,+0) │  (+,+0)
  ───────◎──────→ +X (旋转中心)
  (-,-)  │  (+,-)
```

- **纵向轮 offset**：右侧 → 正，左侧 → 负
- **横向轮 offset**：前方 → 正，后方 → 负

### 航向计算优先级

```
IMU → 横向轮差角 → 纵向轮差角 → 报错
```

> 最佳配置：**1 个 IMU + 1 纵向轮 + 1 横向轮**

## 9. PID 控制器 (PID.cpp)

### 源文件

`src/lemlib/PID.cpp` + `include/lemlib/PID.hpp`

### PID 公式

$$output = e \cdot kP + \int e \, dt \cdot kI + \frac{de}{dt} \cdot kD$$

| 项 | 作用 | 类比 |
|----|------|------|
| **P（比例）** | 现在：误差越大，输出越大 | 踩油门——离目标远就踩深一点 |
| **I（积分）** | 过去：累积误差消除稳态偏差 | 越等越急——时间越长越用力 |
| **D（微分）** | 未来：抑制变化，防止超调 | 预判刹车——靠近目标时提前减速 |

### 类成员变量

```cpp
Gains m_gains;                    // {kP, kI, kD} 增益
Number m_windupRange;             // 抗积分饱和阈值
bool m_signFlipReset;             // 误差符号翻转时是否清零积分
std::optional<Time> m_previousTime; // 上次更新时间（用于 dt）
Number m_previousError;           // 上次误差（用于微分）
Number m_integral;                // 累积积分值
```

### 核心方法：update(error)

```cpp
Number PID::update(Number error) {
    // ① 计算时间间隔 dt（首次调用 dt=0）
    const Time now = from_msec(pros::millis());
    Time dt = (m_previousTime == std::nullopt) ? 0_msec : now - *m_previousTime;
    m_previousTime = now;

    // ② D 项：误差变化率
    const Number derivative = (dt != 0_sec)
        ? (error - m_previousError) / to_sec(dt) : 0;

    // ③ I 项：累积积分
    m_integral += error * to_sec(dt);

    // ④ 符号翻转复位：越过目标时清零积分
    if (sgn(error) != sgn(m_previousError) && m_signFlipReset)
        m_integral = 0;

    // ⑤ 抗积分饱和：误差太大时不积分
    if (abs(error) > m_windupRange && m_windupRange != 0)
        m_integral = 0;

    m_previousError = error;

    // ⑥ 合成输出：P + I + D
    return error * m_gains.kP + m_integral * m_gains.kI + derivative * m_gains.kD;
}
```

### 处理流程说明

| 步骤 | 内容 | 意义 |
|------|------|------|
| ① dt | `now - previousTime` | 计算帧间隔，首次为 0 防脏数据 |
| ② D项 | `Δerror / dt` | 误差变化速度 → 提前刹车 |
| ③ I项 | `Σ(error × dt)` | 累积微小误差 → 克服静摩擦 |
| ④ 翻转复位 | `sgn(error) ≠ sgn(prev)` | 越过目标时清零积分，防 overshoot |
| ⑤ 抗饱和 | `|error| > windupRange` | 大误差时不积分，防过冲后震荡 |
| ⑥ 输出 | `P + I + D` | 三项线性叠加 |

### 其他方法

| 方法 | 作用 |
|------|------|
| `reset()` | 清零积分和上次误差（运动开始时调用） |
| `setGains(gains)` | 运行时修改 kP/kI/kD |
| `getGains()` | 查询当前增益 |
| `setWindupRange(range)` | 设置抗饱和阈值（0 = 禁用） |
| `setSignFlipReset(bool)` | 启用/禁用符号翻转复位 |

### 在运动控制中的位置

```mermaid
flowchart LR
    ODOM["里程计 getPose()"] --> ERR["计算误差 (目标 - 当前位置)"]
    ERR --> PID["PID::update(error)"]
    PID --> CLAMP["clamp + slew (限幅+加速度限制)"]
    CLAMP --> DESAT["desaturate (横向+角向混合)"]
    DESAT --> MOTORS["left/right motors.move()"]
```

> 每次 `moveToPoint`/`moveToPose` 循环（每 10ms），都会调用 `settings.lateralPID.update()` 和 `settings.angularPID.update()` 计算功率。

### 构造函数示例

```cpp
// 完整参数
lemlib::PID pid(10, 0, 100, 10, true);
//              kP  kI kD  windup  signFlip

// 最简（无 anti-windup，无符号翻转复位）

// 用 Gains 结构体
lemlib::Gains gains = {10, 0.5, 50};
lemlib::PID pid(gains, 5, true);
```

## 10. 升降机构 Lift() & 六段式 PID

### 文件

- `include/motorcontrol.h` — PID 结构体 + 函数声明
- `src/motorcontrol.cpp` — 完整实现

### PID 类：`LiftPID`

```cpp
struct LiftPID {
    float kP, kI, kD;
    float output(float error, float lastError, float accError) const {
        return kP * error + kI * accError + kD * (error - lastError);
    }
};
```

与 LemLib `PID` 的区别：

| | LemLib PID | LiftPID |
|---|---|---|
| 数值类型 | `Number` (double) | `float` |
| 状态管理 | 内部自动管理 lastError + integral | 外部手动管理 `last_err[]` + `acc_err[]` |
| 积分处理 | 内部 `m_integral`，含 anti-windup | 由调用方在外部做 anti-windup |
| 时间 dt | 内部自动计算 | 不计算（假定恒定循环周期） |
| 使用场景 | 底盘运动控制 | 子系统（升降/机械臂） |

> **为什么不用 LemLib PID？** 升降机构需要六段不同的 PID 参数（上升 3 段 + 下降 3 段），每段独立管理积分。LemLib PID 的积分内部化不适用于这种多实例交替切换的场景。

### 六段式设计

```
总行程 93°：底 0° ← 359°...267° → 顶
每段 31°

上升 (err < 0)：                下降 (err > 0)：
  段0(底) → LIFT_UP_1   idx=0      段2(顶) → LIFT_DOWN_3 idx=5
  段1(中) → LIFT_UP_2   idx=1      段1(中) → LIFT_DOWN_2 idx=4
  段2(顶) → LIFT_UP_3   idx=2      段0(底) → LIFT_DOWN_1 idx=3
```

方向切换时，另一方向的 3 段积分全部清零。

### 硬件对象

```cpp
static pros::Motor    lift1(11);        // 左侧升降电机
static pros::Motor    lift2(12);        // 右侧升降电机
static pros::Rotation liftRotation(3);  // 编码器
```

### VEXcode → PROS API 对照

| VEXcode | PROS |
|---------|------|
| `Rotation.angle(deg)` | `liftRotation.get_angle() / 100.0f` |
| `lift1.stop(hold)` | `lift1.set_brake_mode(E_MOTOR_BRAKE_HOLD); lift1.move(0);` |
| `lift1.spin(fwd, 0.128 * out, volt)` | `lift1.move_voltage(out * 128)` (毫伏) |
| `print(out, 2)` | `pros::lcd::print(2, "%.2f", out);` |
| `fabs` / `fmod` | `std::fabs` / `std::fmod` (`<cmath>`) |

> ⚠️ PROS Rotation 传感器返回厘度（centidegrees），需 `/100.0` 转为度。
> ⚠️ VEXcode `spin(voltage)` 用伏特，PROS `move_voltage()` 用毫伏，`out * 128` 即 `0.128V × 1000`。

### 关键设计细节

| 机制 | 实现 | 作用 |
|------|------|------|
| 首次锁定 | `target_init` + 静态变量 | 上电不猛冲，锁定当前位置 |
| 粘性边界 | `atTop` / `atBottom` | 到顶推上无效，到底推下无效 |
| 机械限位 | 不可达区间 (0, 267) 夹到边界 | 防止 target 在无效位置 |
| 角度环绕 | `err > 180 → err -= 360` | 取最短路径旋转 |
| 六段积分分离 | 6 个独立 `acc_err[]` | 上升/下降不同 PID，互不干扰 |
| 底部软着陆 | `distFromBottom < 15°` 平滑减速 | 防止到底硬撞击 |
| 到位锁死 | `|err| < 1.4° && |out| < 17` → HOLD | 到位后主动保持位置 |

### 在 opcontrol() 中调用

```cpp
void opcontrol() {
    pros::Controller master(pros::E_CONTROLLER_MASTER);

    while (true) {
        int leftY  = master.get_analog(ANALOG_LEFT_Y);
        int rightX = master.get_analog(ANALOG_RIGHT_X);

        // 底盘 Arcade Drive
        left_mg.move(leftY - rightX);
        right_mg.move(leftY + rightX);

        // 升降机构
        int liftJoystick = master.get_analog(ANALOG_RIGHT_Y);
        Lift(liftJoystick);

        pros::delay(20);
    }
}
```
